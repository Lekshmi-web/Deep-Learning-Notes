HOW RNN WORKS
At every word, the RNN does one thing: ht = tanh(W·xt + U·ht-1)

xt = the current word  ·  ht-1 = memory from the previous step

The problem: each step overwrites the memory a little. After many words, early information is almost gone. This is the vanishing gradient problem.

HOW LSTM FIXES THE PROBLEM
LSTM adds a cell state — a separate "long-term memory highway" that runs alongside the hidden state.

The three gates decide:
🗑 Forget gate — how much of the old memory to erase
✍ Input gate — how much of the new word to write in
📤 Output gate — what to pass to the next step

Important information can survive 20+ steps without vanishing.

RNN
Recurrent Neural Network
✅Simple and fast to train
✅Good for short sequences
❌Forgets early words (vanishing gradient)
❌Struggles with long sentences
Use for: POS tagging, short text, simple time-series
LSTM
Long Short-Term Memory
✅Remembers across long sequences
✅Gates control what to keep/forget
✅Used in speech, translation, sentiment
⚠️Slower — 4× more parameters than RNN
Use for: Translation, sentiment, speech recognition
WHERE ARE THEY TODAY?
Transformers (like GPT, BERT) have replaced RNN/LSTM for most language tasks — they use attention instead of sequential processing, so they don't have the vanishing gradient problem at all.

But LSTM is still widely used for real-time, on-device tasks — like voice assistants, autocomplete on your phone, and stock prediction — where speed and low memory matter.
